    ‚ú® Refactoring Text Editor to Flyweight Pattern

    üéØ Goal

    Reduce memory usage by sharing character style objects instead of duplicating them for every character.

    üî¥ BEFORE (Problem in Existing Design)

    class StyledChar {
    char ch;
    String fontName;
    int fontSize;
    boolean isBold;
    boolean isItalic;
}

    ‚ùå Issue:

    • Every character stores full style info
    • Large document ‚áí huge memory waste
    • Same styles repeated thousands of times

    ‚úÖ AFTER (Flyweight Design)

    We split style (intrinsic, shared) from character data (extrinsic).

    üß± Step 1: Create Flyweight (TextStyle)

    class TextStyle {
    final String fontName;
    final int fontSize;
    final boolean isBold;
    final boolean isItalic;

    TextStyle(String fontName, int fontSize, boolean isBold, boolean isItalic) {
        this.fontName = fontName;
        this.fontSize = fontSize;
        this.isBold = isBold;
        this.isItalic = isItalic;
    }

    @Override
    public boolean equals(Object o) {
        if (this == o) return true;
        if (!(o instanceof TextStyle)) return false;
        TextStyle that = (TextStyle) o;
        return fontSize == that.fontSize &&
               isBold == that.isBold &&
               isItalic == that.isItalic &&
               fontName.equals(that.fontName);
    }

    @Override
    public int hashCode() {
        return Objects.hash(fontName, fontSize, isBold, isItalic);
    }
}

    ‚úÖ This is the Flyweight object ‚úÖ Must override equals() & hashCode() ‚úÖ Immutable ‚Üí safe to share

    üè≠ Step 2: Flyweight Factory

    class TextStyleFactory {

    private static final Map<TextStyle, TextStyle> cache = new HashMap<>();

    public static TextStyle getStyle(
            String fontName,
            int fontSize,
            boolean isBold,
            boolean isItalic) {

        TextStyle key = new TextStyle(fontName, fontSize, isBold, isItalic);
        cache.putIfAbsent(key, key);
        return cache.get(key);
    }
}

    ‚úî Ensures only one object per unique style ‚úî Centralized style reuse

    ‚úçÔ∏è Step 3: Character Holds Only What‚Äôs Unique

    class StyledChar {
    char ch;              // extrinsic
    TextStyle style;      // intrinsic (shared)

    StyledChar(char ch, TextStyle style) {
        this.ch = ch;
        this.style = style;
    }
}

    üìÑ Step 4: Document Storage (Rows)

    List<List<StyledChar>> rows = new ArrayList<>();

    ‚úèÔ∏è Refactored addCharacter() (Flyweight Enabled)

    public void addCharacter(
        int row, int column, char ch,
        String fontName, int fontSize,
        boolean isBold, boolean isItalic) {

    while (rows.size() <= row) {
        rows.add(new ArrayList<>());
    }

    List<StyledChar> line = rows.get(row);

    TextStyle style = TextStyleFactory.getStyle(
            fontName, fontSize, isBold, isItalic);

    StyledChar newChar = new StyledChar(ch, style);

    if (column >= line.size()) {
        line.add(newChar);
    } else {
        line.add(column, newChar);
    }
}

    üí° Key improvement: Style objects are shared automatically.

    üé® Refactored getStyle()

    public String getStyle(int row, int col) {
    if (row >= rows.size() || col >= rows.get(row).size()) {
        return "";
    }

    StyledChar sc = rows.get(row).get(col);
    TextStyle s = sc.style;

    StringBuilder sb = new StringBuilder();
    sb.append(sc.ch).append("-")
      .append(s.fontName).append("-")
      .append(s.fontSize);

    if (s.isBold) sb.append("-b");
    if (s.isItalic) sb.append("-i");

    return sb.toString();
}

    üìñ readLine() (Unchanged)

    public String readLine(int row) {
    if (row >= rows.size()) return "";

    StringBuilder sb = new StringBuilder();
    for (StyledChar sc : rows.get(row)) {
        sb.append(sc.ch);
    }
    return sb.toString();
}

    üóëÔ∏è deleteCharacter() (Unchanged)

    public boolean deleteCharacter(int row, int col) {
    if (row >= rows.size() || col >= rows.get(row).size()) {
        return false;
    }
    rows.get(row).remove(col);
    return true;
}

    üìä Memory Impact (Why This Matters)

    Scenario Without Flyweight With Flyweight
    1M chars, same style 1M style objects 1 style object
    Real editor ‚ùå Heavy ‚úÖ Efficient
    Interview quality ‚ùå Average üî• Strong

    ‚ÄúI used the Flyweight pattern to deduplicate text formatting objects. Each character stores only a reference to a shared TextStyle, drastically reducing memory usage for large documents.‚Äù
