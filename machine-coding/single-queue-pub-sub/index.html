    33. Design a Single-Queue Publish Subscribe System

    üìú Problem Statement

    Design an in-memory publish/subscribe system with exactly one global FIFO queue.

    • Multiple publishers can publish messages
    • Multiple subscribers can subscribe to the same queue
    • Every message is appended to a single global FIFO queue
    • All active subscribers are notified on publish

    • Each subscriber:

      • Consumes messages independently
      • Processes only selected event types
      • Sees only messages published while it was subscribed

    üìå Important

    • No persistence
    • No retroactive delivery
    • Single queue only
    • Use Observer Pattern

    üß† Core Rules

    • Subscribers filter messages by eventType
    • Ignored event types do not count as processed

    • A subscriber may:

      • Subscribe
      • Unsubscribe
      • Re-subscribe with a new filter
      • Processed message counts must be cumulative across sessions

    üß© APIs to Implement

    1Ô∏è‚É£ addSubscriber

    void addSubscriber(String subscriberId, List<String> eventTypesToProcess)
    • Registers a subscriber
    • Re-adding replaces old filters
    • No retroactive message delivery

    2Ô∏è‚É£ removeSubscriber

    void removeSubscriber(String subscriberId)
    • Stops further message delivery
    • Preserves processed count so far

    3Ô∏è‚É£ sendMessage

    void sendMessage(String eventType, String message)
    • Appends message to the global FIFO queue
    • Notifies all active subscribers
    • Each subscriber decides whether to process or ignore

    4Ô∏è‚É£ countProcessedMessages

    int countProcessedMessages(String subscriberId)
    • Returns total messages actually processed

    • Includes counts from:

      • Previous subscriptions
      • Current active session (if any)

    üìò Example Walkthrough

    sendMessage("ORDER", "pre-1");   // no subscribers ‚Üí nobody processes

addSubscriber("S1", ["ORDER","PAYMENT"]);
addSubscriber("S2", ["PAYMENT"]);

sendMessage("ORDER", "o-1");     // S1 processes, S2 ignores

addSubscriber("S3", ["SHIPMENT"]);

sendMessage("PAYMENT", "p-1");   // S1 & S2 process

sendMessage("SHIPMENT", "s-1");  // S3 processes

removeSubscriber("S2");

sendMessage("PAYMENT", "p-2");   // only S1 processes

removeSubscriber("S1");

sendMessage("ORDER", "o-2");     // nobody processes

addSubscriber("S1", ["ORDER","REFUND"]);

sendMessage("REFUND", "r-1");    // S1 processes

    Processed Counts

    countProcessedMessages("S1") ‚Üí 4
countProcessedMessages("S2") ‚Üí 1
countProcessedMessages("S3") ‚Üí 1

    üèóÔ∏è Design Overview

    Key Components

    Component Responsibility
    Message Event payload with type
    Subscriber Observer interface
    ConcreteSubscriber Applies filtering + counts
    SingleQueuePubSubSystem Subject / Queue Manager

    üß† Design Pattern Used

    ‚úÖ Observer Pattern

    • Subject: Queue Manager
    • Observers: Subscribers

    • Enables:

      • Loose coupling
      • Dynamic subscription
      • Independent consumption pace

    üíª Java Implementation

    import java.util.*;

public class SingleQueuePubSubSystem {

    class Message {
        String eventType;
        String payload;

        Message(String eventType, String payload) {
            this.eventType = eventType;
            this.payload = payload;
        }
    }

    interface Subscriber {
        void onMessage(Message message);
    }

    class ConcreteSubscriber implements Subscriber {

        String subscriberId;
        Set<String> eventTypes;
        int processedCount = 0;

        ConcreteSubscriber(String subscriberId, List<String> eventTypes) {
            this.subscriberId = subscriberId;
            this.eventTypes = new HashSet<>(eventTypes);
        }

        @Override
        public void onMessage(Message message) {
            if (eventTypes.contains(message.eventType)) {
                processedCount++;
            }
        }
    }

    private List<Message> queue = new ArrayList<>();
    private Map<String, ConcreteSubscriber> activeSubscribers = new HashMap<>();
    private Map<String, Integer> totalProcessedCounts = new HashMap<>();

    public void addSubscriber(String subscriberId, List<String> eventTypesToProcess) {

        if (activeSubscribers.containsKey(subscriberId)) {
            removeSubscriber(subscriberId);
        }

        ConcreteSubscriber sub =
                new ConcreteSubscriber(subscriberId, eventTypesToProcess);

        activeSubscribers.put(subscriberId, sub);
    }

    public void removeSubscriber(String subscriberId) {

        ConcreteSubscriber sub = activeSubscribers.remove(subscriberId);
        if (sub == null) return;

        int prev = totalProcessedCounts.getOrDefault(subscriberId, 0);
        totalProcessedCounts.put(subscriberId, prev + sub.processedCount);
    }

    public void sendMessage(String eventType, String message) {

        Message msg = new Message(eventType, message);
        queue.add(msg);

        for (ConcreteSubscriber sub : activeSubscribers.values()) {
            sub.onMessage(msg);
        }
    }

    public int countProcessedMessages(String subscriberId) {

        int total = totalProcessedCounts.getOrDefault(subscriberId, 0);

        ConcreteSubscriber active = activeSubscribers.get(subscriberId);
        if (active != null) {
            total += active.processedCount;
        }

        return total;
    }
}

    ‚ö†Ô∏è Edge Cases Covered

    • Late subscribers (no retroactive delivery)
    • Event type filtering
    • Unsubscribe immediately stops processing
    • Re-subscribe with new filters
    • Messages with no eligible subscribers
    • Cumulative processed counts

    Operation Time
    sendMessage O(S)
    addSubscriber O(1)
    removeSubscriber O(1)
    countProcessedMessages O(1)

    Where S = active subscribers
