    25. Design a Spreadsheet (Excel-like)

    üìú Problem Statement

    Design an in-memory spreadsheet system similar to Microsoft Excel.

    The spreadsheet supports:

    • Dynamic insertion of rows and columns
    • Each cell storing text + style
    • Deterministic access to any cell
    • Single-threaded execution

    üî¢ Initialization Rules

    • Initial size: 5 rows √ó 5 columns
    • Rows and columns are 0-indexed
    • All cells start empty
    • Empty cell returns an empty string

    üß† Design Overview

    Core Ideas

    • Spreadsheet modeled as a 2D dynamic structure

    • Each cell encapsulates:

      • text
      • font name
      • font size
      • bold / italic flags
      • Insertions shift existing rows/columns automatically

    üß± Data Model

    Cell Structure

    class Cell {
    String text;
    String fontName;
    int fontSize;
    boolean isBold;
    boolean isItalic;

    Cell(String text, String fontName, int fontSize,
         boolean isBold, boolean isItalic) {
        this.text = text;
        this.fontName = fontName;
        this.fontSize = fontSize;
        this.isBold = isBold;
        this.isItalic = isItalic;
    }
}

    üèóÔ∏è Complete Java Implementation

    import java.util.*;

public class Spreadsheet {

    private List<List<Cell>> sheet;
    private int rows;
    private int cols;

    // ---------- Initialization ----------
    public Spreadsheet() {
        rows = 5;
        cols = 5;
        sheet = new ArrayList<>();

        for (int i = 0; i < rows; i++) {
            List<Cell> row = new ArrayList<>(Collections.nCopies(cols, null));
            sheet.add(row);
        }
    }

    // ---------- Add Row ----------
    public void addRow(int index) {
        if (index < 0 || index > rows) return;

        List<Cell> newRow = new ArrayList<>(Collections.nCopies(cols, null));
        sheet.add(index, newRow);
        rows++;
    }

    // ---------- Add Column ----------
    public void addColumn(int index) {
        if (index < 0 || index > cols) return;

        for (List<Cell> row : sheet) {
            row.add(index, null);
        }
        cols++;
    }

    // ---------- Add / Replace Entry ----------
    public void addEntry(int row, int column, String text,
                         String fontName, int fontSize,
                         boolean isBold, boolean isItalic) {

        if (row < 0 || row >= rows || column < 0 || column >= cols) return;

        sheet.get(row).set(
                column,
                new Cell(text, fontName, fontSize, isBold, isItalic)
        );
    }

    // ---------- Get Entry ----------
    public String getEntry(int row, int column) {
        if (row < 0 || row >= rows || column < 0 || column >= cols) {
            return "";
        }

        Cell cell = sheet.get(row).get(column);
        if (cell == null) return "";

        StringBuilder sb = new StringBuilder();
        sb.append(cell.text)
          .append("-")
          .append(cell.fontName)
          .append("-")
          .append(cell.fontSize);

        if (cell.isBold) sb.append("-b");
        if (cell.isItalic) sb.append("-i");

        return sb.toString();
    }
}

    üìå Method Behavior Summary

    addRow(index)

    • Inserts a new empty row at the given index
    • Existing rows shift downward

    addColumn(index)

    • Inserts a new empty column at the given index
    • Existing columns shift right

    addEntry(row, column, ...)

    • Creates or replaces a cell
    • Overwrites any existing cell at the location

    getEntry(row, column)

    • Returns formatted string:

    text-fontName-fontSize[-b][-i]
    * Returns empty string if cell is empty

    ‚úÖ Example Walkthrough

    addRow(5);
addEntry(5, 0, "hello", "tahoma", 24, true, false);
getEntry(5, 0); 
// "hello-tahoma-24-b"

addColumn(2);
addEntry(0, 1, "x", "calibri", 10, false, true);
getEntry(0, 1);
// "x-calibri-10-i"

    ‚è±Ô∏è Time & Space Complexity

    Operation Complexity
    addRow O(columns)
    addColumn O(rows)
    addEntry O(1)
    getEntry O(1)
    Space O(rows √ó columns)

    üß† Design Patterns (Interview Bonus)

    • Flyweight Pattern Share identical style objects to reduce memory usage
    • Command Pattern Enables undo/redo functionality
    • MVC Architecture Separate data model from UI rendering
