    ‚öôÔ∏è Design a Job Scheduler

    üìú Problem Statement

    Design a Job Scheduler for a massively parallel distributed system.

    • The scheduler manages multiple machines
    • Each machine has a set of capabilities
    • Each job requires a set of capabilities
    • A job can only be assigned to a machine that has all required capabilities

    üß† Core Requirements

    Machine

    • Has a unique machineId
    • Has a set of capabilities (case-insensitive)

    • Tracks:

      • unfinishedJobs
      • finishedJobs

    Job

    • Has a unique jobId
    • Assigned to exactly one machine
    • Can be marked completed

    üîÄ Assignment Criteria (Extensible)

    When multiple machines qualify, select based on criteria:

    Criteria Description
    0 Machine with least unfinished jobs
    1 Machine with most finished jobs
    Tie Lexicographically smallest machineId

    ‚ö†Ô∏è Extensibility requirement: Adding a new scheduling algorithm should not change existing logic.

    üèóÔ∏è Design Overview

    Key Design Patterns Used

    Pattern Why
    Strategy Pattern Plug-and-play job selection algorithms
    Encapsulation Machine manages its own counters
    Open/Closed Principle Add new criteria without touching scheduler logic

    üíª Java Implementation

    Assignment Strategy Interface

    interface AssignmentStrategy {
    Machine select(List<Machine> candidates);
}

    Strategy 0: Least Unfinished Jobs

    class LeastUnfinishedStrategy implements AssignmentStrategy {

    @Override
    public Machine select(List<Machine> machines) {
        machines.sort((a, b) -> {
            if (a.unfinishedJobs != b.unfinishedJobs) {
                return a.unfinishedJobs - b.unfinishedJobs;
            }
            return a.machineId.compareTo(b.machineId);
        });
        return machines.get(0);
    }
}

    Strategy 1: Most Finished Jobs

    class MostFinishedStrategy implements AssignmentStrategy {

    @Override
    public Machine select(List<Machine> machines) {
        machines.sort((a, b) -> {
            if (a.finishedJobs != b.finishedJobs) {
                return b.finishedJobs - a.finishedJobs;
            }
            return a.machineId.compareTo(b.machineId);
        });
        return machines.get(0);
    }
}

    Machine

    class Machine {
    String machineId;
    Set<String> capabilities;
    int unfinishedJobs = 0;
    int finishedJobs = 0;

    Machine(String machineId, Set<String> capabilities) {
        this.machineId = machineId;
        this.capabilities = capabilities;
    }

    boolean supports(Set<String> required) {
        return capabilities.containsAll(required);
    }
}

    Job

    class Job {
    String jobId;
    Machine machine;

    Job(String jobId, Machine machine) {
        this.jobId = jobId;
        this.machine = machine;
    }
}

    Job Scheduler (Main Class)

    import java.util.*;

public class JobScheduler {

    private Map<String, Machine> machines = new HashMap<>();
    private Map<String, Job> jobs = new HashMap<>();
    private Map<Integer, AssignmentStrategy> strategies = new HashMap<>();

    public JobScheduler() {
        strategies.put(0, new LeastUnfinishedStrategy());
        strategies.put(1, new MostFinishedStrategy());
    }

    // ---------------- addMachine ----------------
    public void addMachine(String machineId, String[] capabilities) {
        Set<String> caps = new HashSet<>();
        for (String cap : capabilities) {
            caps.add(cap.toLowerCase());
        }
        machines.put(machineId, new Machine(machineId, caps));
    }

    // ---------------- assignMachineToJob ----------------
    public String assignMachineToJob(String jobId, String[] requiredCaps, int criteria) {

        Set<String> required = new HashSet<>();
        for (String r : requiredCaps) {
            required.add(r.toLowerCase());
        }

        List<Machine> candidates = new ArrayList<>();
        for (Machine m : machines.values()) {
            if (m.supports(required)) {
                candidates.add(m);
            }
        }

        if (candidates.isEmpty()) return "";

        AssignmentStrategy strategy = strategies.get(criteria);
        Machine selected = strategy.select(candidates);

        selected.unfinishedJobs++;
        jobs.put(jobId, new Job(jobId, selected));

        return selected.machineId;
    }

    // ---------------- jobCompleted ----------------
    public void jobCompleted(String jobId) {
        Job job = jobs.get(jobId);
        Machine m = job.machine;

        m.unfinishedJobs--;
        m.finishedJobs++;
    }
}

    üß™ Example Walkthrough

    addMachine("m-10", ["image compression", "audio extraction"])
addMachine("m-2", ["image compression"])

assignMachineToJob("job-A", ["image compression"], 0)
‚Üí "m-10" (tie ‚Üí lexicographically smaller)

jobCompleted("job-A")

assignMachineToJob("job-B", ["image compression"], 1)
‚Üí "m-10" (most finished jobs)

    ‚ö†Ô∏è Edge Cases Handled

    • Case-insensitive capabilities
    • No compatible machines ‚Üí return ""
    • Deterministic tie-breaking
    • Multiple jobs per machine
    • Strategy extensibility without refactoring

    ‚è±Ô∏è Complexity Analysis

    Operation Complexity
    addMachine O(c)
    assignMachineToJob O(m log m)
    jobCompleted O(1)
    Space O(m + j + c)

    Where:

    • m = machines
    • j = jobs
    • c = capabilities
