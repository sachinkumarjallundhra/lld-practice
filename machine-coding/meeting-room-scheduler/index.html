    44. Design a Meeting Room Scheduler ‚Äì List Bookings

    üìò Problem Statement

    Design an in-memory Meeting Room Scheduler that allows employees to:

    • View available rooms
    • Book a room for a time interval
    • Cancel a booking
    • List bookings by room or by employee

    Core Rules

    • Rooms are identified by integers: 0 to roomsCount - 1
    • Employees are identified by integers: 0 to employeesCount - 1
    • A booking reserves exactly one room for exactly one employee

    • Time intervals are closed: [startTime, endTime]

      • Example: [10,20] and [20,30] overlap
      • No double booking: a room cannot have overlapping bookings

    üß† Design Overview

    Entities

    • Booking
    • Room
    • Employee
    • Scheduler (Coordinator)

    Key Design Decisions

    • Store bookings:

      • By bookingId
      • Grouped by roomId
      • Grouped by employeeId
      • Use inclusive interval overlap check
      • Sorting done only at read time (not on every insert)

    üß± Data Structures Used

    Purpose Data Structure
    Lookup booking by ID Map<String, Booking>
    Bookings per room Map<Integer, List<Booking>>
    Bookings per employee Map<Integer, List<Booking>>

    üß© Class Design

    Booking

    class Booking {
    String bookingId;
    int employeeId;
    int roomId;
    int start;
    int end;
}

    üîå API Methods

    Constructor

    MeetingRoomScheduler(int roomsCount, int employeesCount)

    Initializes scheduler with fixed rooms and employees.

    Book Room

    boolean bookRoom(String bookingId, int employeeId,
                 int roomId, int startTime, int endTime)

    Rules

    • Invalid if startTime > endTime or startTime < 0
    • Reject if any overlap exists in the same room
    • Atomic: either booking is fully created or nothing happens

    Get Available Rooms

    List<Integer> getAvailableRooms(int startTime, int endTime)
    • Returns rooms free for the entire interval
    • Invalid range ‚Üí empty list
    • Returned in ascending order

    Cancel Booking

    boolean cancelBooking(String bookingId)

    • Removes booking from:

      • booking map
      • room list
      • employee list

    List Bookings for Room

    List<String> listBookingsForRoom(int roomId)

    Sorted by:

    1. startTime
    2. bookingId (lexicographically)

    List Bookings for Employee

    List<String> listBookingsForEmployee(int employeeId)

    Same sorting rule as above.

    ‚ö†Ô∏è Interval Overlap Logic (Critical)

    Closed interval overlap condition:

    overlap if max(start1, start2) <= min(end1, end2)

    üíª Complete Java Implementation

    import java.util.*;

public class MeetingRoomScheduler {

    static class Booking {
        String bookingId;
        int employeeId;
        int roomId;
        int start;
        int end;

        Booking(String bookingId, int employeeId,
                int roomId, int start, int end) {
            this.bookingId = bookingId;
            this.employeeId = employeeId;
            this.roomId = roomId;
            this.start = start;
            this.end = end;
        }
    }

    private Map<String, Booking> bookingById = new HashMap<>();
    private Map<Integer, List<Booking>> roomBookings = new HashMap<>();
    private Map<Integer, List<Booking>> employeeBookings = new HashMap<>();

    private int roomsCount;
    private int employeesCount;

    public MeetingRoomScheduler(int roomsCount, int employeesCount) {
        this.roomsCount = roomsCount;
        this.employeesCount = employeesCount;

        for (int i = 0; i < roomsCount; i++) {
            roomBookings.put(i, new ArrayList<>());
        }
        for (int i = 0; i < employeesCount; i++) {
            employeeBookings.put(i, new ArrayList<>());
        }
    }

    public boolean bookRoom(String bookingId, int employeeId,
                            int roomId, int startTime, int endTime) {

        if (startTime < 0 || startTime > endTime) return false;

        for (Booking b : roomBookings.get(roomId)) {
            if (overlaps(b.start, b.end, startTime, endTime)) {
                return false;
            }
        }

        Booking booking = new Booking(
            bookingId, employeeId, roomId, startTime, endTime
        );

        bookingById.put(bookingId, booking);
        roomBookings.get(roomId).add(booking);
        employeeBookings.get(employeeId).add(booking);

        return true;
    }

    public List<Integer> getAvailableRooms(int startTime, int endTime) {
        List<Integer> result = new ArrayList<>();
        if (startTime > endTime) return result;

        for (int roomId = 0; roomId < roomsCount; roomId++) {
            boolean free = true;
            for (Booking b : roomBookings.get(roomId)) {
                if (overlaps(b.start, b.end, startTime, endTime)) {
                    free = false;
                    break;
                }
            }
            if (free) result.add(roomId);
        }
        return result;
    }

    public boolean cancelBooking(String bookingId) {
        Booking booking = bookingById.remove(bookingId);
        if (booking == null) return false;

        roomBookings.get(booking.roomId).remove(booking);
        employeeBookings.get(booking.employeeId).remove(booking);
        return true;
    }

    public List<String> listBookingsForRoom(int roomId) {
        List<Booking> list =
            new ArrayList<>(roomBookings.get(roomId));
        sortBookings(list);

        List<String> res = new ArrayList<>();
        for (Booking b : list) res.add(b.bookingId);
        return res;
    }

    public List<String> listBookingsForEmployee(int employeeId) {
        List<Booking> list =
            new ArrayList<>(employeeBookings.get(employeeId));
        sortBookings(list);

        List<String> res = new ArrayList<>();
        for (Booking b : list) res.add(b.bookingId);
        return res;
    }

    // ---------- Helpers ----------

    private boolean overlaps(int s1, int e1, int s2, int e2) {
        return Math.max(s1, s2) <= Math.min(e1, e2);
    }

    private void sortBookings(List<Booking> list) {
        Collections.sort(list, (a, b) -> {
            if (a.start != b.start)
                return a.start - b.start;
            return a.bookingId.compareTo(b.bookingId);
        });
    }
}

    ‚è±Ô∏è Complexity Analysis

    Operation Complexity
    Book room O(B) per room
    Cancel booking O(B)
    List bookings O(B log B)
    Get availability O(R √ó B)

    B = bookings per room, R = rooms count

    üéØ Interview Takeaways

    • Correct handling of closed intervals
    • Clean separation of indices (by room, by employee)

    • Easy to extend for:

      • Capacity-based rooms
      • Recurring meetings
      • Priority employees
