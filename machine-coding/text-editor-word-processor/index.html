    üìù Design a Text Editor / Word Processor (Microsoft Word)

    üìú Problem Statement

    Design an in-memory text editor / word processor similar to Microsoft Word or WordPad.

    The document:

    • Has any number of rows
    • Each row has any number of columns
    • Each character has its own style

    Character Style Attributes

    • Font Name
    • Font Size
    • Bold
    • Italic

    The system must support:

    • Adding characters
    • Deleting characters
    • Reading rows
    • Fetching style of a specific character

    üß† Key Design Decisions

    1Ô∏è‚É£ Row-based storage

    • Each row is stored independently
    • Rows are dynamically created

    2Ô∏è‚É£ Character-level styling

    • Each character is wrapped in a StyledChar object
    • Style stays bound to the character even after shifts

    3Ô∏è‚É£ Insert & Delete via shifting

    • Insert ‚Üí shift characters to the right
    • Delete ‚Üí shift characters to the left

    üíª Java Implementation

    import java.util.*;

public class Solution {

    // Helper is provided by the platform
    private Helper09 helper;

    // Each row is a list of styled characters
    private List<List<StyledChar>> document = new ArrayList<>();

    // ---------------- init ----------------
    public void init(Helper09 helper) {
        this.helper = helper;
    }

    // ---------------- Styled Character ----------------
    static class StyledChar {
        char ch;
        String fontName;
        int fontSize;
        boolean bold;
        boolean italic;

        StyledChar(char ch, String fontName, int fontSize, boolean bold, boolean italic) {
            this.ch = ch;
            this.fontName = fontName;
            this.fontSize = fontSize;
            this.bold = bold;
            this.italic = italic;
        }

        String styleString() {
            StringBuilder sb = new StringBuilder();
            sb.append(ch)
              .append("-")
              .append(fontName)
              .append("-")
              .append(fontSize);

            if (bold) sb.append("-b");
            if (italic) sb.append("-i");

            return sb.toString();
        }
    }

    // ---------------- addCharacter ----------------
    public void addCharacter(
            int row, int column, char ch,
            String fontName, int fontSize,
            boolean isBold, boolean isItalic) {

        // Ensure enough rows exist
        while (document.size() <= row) {
            document.add(new ArrayList<>());
        }

        List<StyledChar> currentRow = document.get(row);
        StyledChar newChar = new StyledChar(ch, fontName, fontSize, isBold, isItalic);

        // Insert or append
        if (column >= currentRow.size()) {
            currentRow.add(newChar);
        } else {
            currentRow.add(column, newChar);
        }
    }

    // ---------------- getStyle ----------------
    public String getStyle(int row, int col) {
        if (row >= document.size()) return "";

        List<StyledChar> currentRow = document.get(row);
        if (col >= currentRow.size()) return "";

        return currentRow.get(col).styleString();
    }

    // ---------------- readLine ----------------
    public String readLine(int row) {
        if (row >= document.size()) return "";

        StringBuilder sb = new StringBuilder();
        for (StyledChar sc : document.get(row)) {
            sb.append(sc.ch);
        }
        return sb.toString();
    }

    // ---------------- deleteCharacter ----------------
    public boolean deleteCharacter(int row, int col) {
        if (row >= document.size()) return false;

        List<StyledChar> currentRow = document.get(row);
        if (col >= currentRow.size()) return false;

        currentRow.remove(col);
        return true;
    }
}

    ‚ö†Ô∏è Edge Cases Handled

    • Adding characters beyond current rows
    • Adding characters beyond current column length
    • Reading empty rows
    • Deleting non-existing characters
    • Style preserved correctly after insert/delete shifts

    ‚è±Ô∏è Complexity Analysis

    Operation Time Complexity
    addCharacter O(n) (row shift)
    deleteCharacter O(n) (row shift)
    getStyle O(1)
    readLine O(n)

    Space Complexity: O(total characters)

    üéØ Interview Notes

    • This is a Flyweight-friendly design (styles can be shared later)
    • Clean separation of data + behavior

    • Easily extensible for:

      • Undo / Redo
      • Copy / Paste
      • Rich formatting blocks
      • Paragraph styles
