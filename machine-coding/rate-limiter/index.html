    34. Design a Rate Limiter

    üìú Problem Statement

    Design an in-memory rate limiter that controls how frequently requests are allowed for different resources.

    Each resourceId has its own rate limiting strategy and limits. The rate limiter must support pluggable strategies so that new algorithms can be added easily.

    Assume 1 time unit = 1 second

    üß† Supported Strategies

    1Ô∏è‚É£ Fixed Window Counter

    • Divides time into fixed-size windows (e.g., every 5 seconds)
    • Tracks request count per window
    • Blocks requests if count exceeds limit
    • ‚úÖ Fast and simple
    • ‚ùå Allows burst at window boundaries

    2Ô∏è‚É£ Sliding Window Counter (Log-Based)

    • Maintains timestamps of recent requests
    • Cleans up timestamps outside the window
    • More accurate enforcement
    • ‚ùå Higher memory & processing cost

    üìê APIs to Implement

    addResource

    void addResource(String resourceId, String strategy, String limits)
    • Registers or updates a resource
    • strategy ‚àà { "fixed-window-counter", "sliding-window-counter" }

    • limits format: "maxRequests,timePeriod"

      • Example: "5,2" ‚Üí max 5 requests every 2 seconds
      • Calling again with same resourceId replaces old strategy & state

    isAllowed

    boolean isAllowed(String resourceId, int timestamp)
    • Returns true if request is allowed
    • Returns false otherwise
    • timestamp is monotonically increasing across all resources

    üß™ Example Walkthrough

    Step 1 ‚Äì Configure Fixed Window

    addResource("login-api", "fixed-window-counter", "2,5");

    Windows: [0..4], [5..9], ‚Ä¶

    isAllowed("login-api", 1) ‚Üí true   // count = 1
isAllowed("login-api", 2) ‚Üí true   // count = 2
isAllowed("login-api", 4) ‚Üí false  // count = 3 > 2

    Step 2 ‚Äì Switch to Sliding Window

    addResource("login-api", "sliding-window-counter", "2,3");

    Old state is discarded.

    isAllowed("login-api", 6) ‚Üí true   // window [4..6] ‚Üí {6}
isAllowed("login-api", 7) ‚Üí true   // window [5..7] ‚Üí {6,7}
isAllowed("login-api", 8) ‚Üí false  // window [6..8] ‚Üí {6,7,8}

    üèóÔ∏è Design Overview

    Key Design Goals

    • Strategy-based extensibility
    • Per-resource isolation
    • Simple in-memory data structures
    • Deterministic behavior

    üß† Design Pattern Used

    ‚úÖ Strategy Pattern

    • Context: RateLimiter
    • Strategy Interface: RateLimitStrategy

    • Concrete Strategies:

      • FixedWindowCounterStrategy
      • SlidingWindowCounterStrategy

    üìå Adding a new algorithm only requires:

    • Implementing RateLimitStrategy
    • Wiring it in addResource

    üíª Java Implementation

    import java.util.*;

public class RateLimiter {

    interface RateLimitStrategy {
        boolean isAllowed(int timestamp);
    }

    // ---------------- Fixed Window Counter ----------------
    class FixedWindowCounterStrategy implements RateLimitStrategy {

        private int maxRequests;
        private int windowSize;

        private int currentWindowStart = -1;
        private int count = 0;

        FixedWindowCounterStrategy(int maxRequests, int windowSize) {
            this.maxRequests = maxRequests;
            this.windowSize = windowSize;
        }

        @Override
        public boolean isAllowed(int timestamp) {

            int windowStart = (timestamp / windowSize) * windowSize;

            if (windowStart != currentWindowStart) {
                currentWindowStart = windowStart;
                count = 0;
            }

            if (count < maxRequests) {
                count++;
                return true;
            }
            return false;
        }
    }

    // ---------------- Sliding Window Counter ----------------
    class SlidingWindowCounterStrategy implements RateLimitStrategy {

        private int maxRequests;
        private int windowSize;
        private Deque<Integer> timestamps = new ArrayDeque<>();

        SlidingWindowCounterStrategy(int maxRequests, int windowSize) {
            this.maxRequests = maxRequests;
            this.windowSize = windowSize;
        }

        @Override
        public boolean isAllowed(int timestamp) {

            int windowStart = timestamp - windowSize + 1;

            while (!timestamps.isEmpty() && timestamps.peekFirst() < windowStart) {
                timestamps.pollFirst();
            }

            if (timestamps.size() < maxRequests) {
                timestamps.addLast(timestamp);
                return true;
            }
            return false;
        }
    }

    // ---------------- RateLimiter ----------------
    private Map<String, RateLimitStrategy> resourceStrategies = new HashMap<>();

    public RateLimiter() {}

    public void addResource(String resourceId, String strategy, String limits) {

        String[] parts = limits.split(",");
        int maxRequests = Integer.parseInt(parts[0]);
        int timePeriod = Integer.parseInt(parts[1]);

        RateLimitStrategy rateLimitStrategy;

        if (strategy.equals("fixed-window-counter")) {
            rateLimitStrategy =
                new FixedWindowCounterStrategy(maxRequests, timePeriod);

        } else if (strategy.equals("sliding-window-counter")) {
            rateLimitStrategy =
                new SlidingWindowCounterStrategy(maxRequests, timePeriod);

        } else {
            throw new IllegalArgumentException("Unknown strategy");
        }

        resourceStrategies.put(resourceId, rateLimitStrategy);
    }

    public boolean isAllowed(String resourceId, int timestamp) {
        return resourceStrategies.get(resourceId).isAllowed(timestamp);
    }
}

    ‚ö†Ô∏è Edge Cases Handled

    • Strategy replacement resets state
    • Per-resource isolation
    • Accurate sliding window cleanup
    • Burst control differences clearly defined

    ‚è±Ô∏è Complexity Analysis

    Strategy Time Space
    Fixed Window O(1) O(1)
    Sliding Window O(k) O(k)

    Where k = maxRequests per resource
